# 看護師休暇希望申請システム - マルチ病院対応版（MVP v2.0）

日本の複数の病院向けの看護師休暇希望申請管理システムです。共用PCでの利用と、個人デバイスからのアクセスを想定したWebアプリケーションです。

🆕 **マルチ病院対応版**: 複数の病院が同一システムを利用できるセキュアな設計に対応しました。病院コードによる完全なデータ分離を実現しています。

## 📋 概要

このシステムは、以下の機能を備えたMVPです：

- **🆕 マルチ病院対応**: 複数の病院が安全にシステムを共有利用可能
- **認証システム**: 病院コード＋IDとPIN（4桁）によるセキュアなログイン
- **🆕 データ完全分離**: 病院コードごとにすべてのデータを完全に分離・管理  
- **有給休暇の自動計算**: 入職日から経験年数を自動計算し、有給日数を付与
- **カレンダー機能**: 月次の休暇希望を「希望休」または「有給」で申請
- **管理画面**: 管理者向けの有給マスタ設定（経験年数ごとの付与日数）
- **プライバシー保護**: スタッフ画面では他人の名前を非表示（人数のみ表示）

## 🆕 マルチ病院対応機能の詳細

### 病院コード（Hospital Code）
- **必須フィールド**: ログイン・登録時に必ず入力、すべてのデータに紐付け
- **フォーマット**: 英文字と数字の組み合わせで3〜20文字（例: HOSP001, Tokyo01など）
- **セキュリティ**: 病院ごとのユーザー、休暇希望、有給マスタを完全に分離

### データ分離のセキュリティ
- ✅ **ユーザーデータ**: 病院コードが異なれば、同じIDでも別ユーザーとして登録可能
- ✅ **休暇申請**: 他の病院のデータは表示されない（同じ病院コード内のみ表示）
- ✅ **有給マスタ**: 各病院が独立したマスタを設定・管理可能
- ✅ **管理者**: 病院コードを入力することで、その病院のみの管理権を取得

### 複数病院での利用フロー
1. 各病院に固有の病院コードを発行（例: HOSP001, HOSP002）
2. スタッフが「病院コード」を入力してログイン/登録
3. 同一の病院コード内のスタッフのみデータを共有
4. 異なる病院コードのスタッフデータは完全に見えない

## 📋 主要機能の仕様

### 1. マルチ病院対応認証システム

#### ログイン
- **病院コード＋ID＋PIN（4桁）** でログイン
- 病院コード ＋ ID で一意の組み合わせを管理
- **同じIDでも病院コードが異なれば別アカウント**

#### 新規登録
- 病院コード、ユーザーID、氏名、PIN（4桁）、入職日を設定
- **同じ病院内でIDの重複をチェック**（他の病院では同じIDでも登録可能）
- 入職日から自動的に経験年数を計算し、有給日数を付与

### 2. 有給休暇の自動計算（病院ごと）

#### マスタ設定（管理者のみ）
- **各病院ごとに、** 経験年数（0年〜6年以上）ごとの年間付与日数を設定可能
- デフォルト値（全病院共通）：
  - 0年（新入職）: 10日
  - 1年: 10日
  - 2年: 11日
  - 3年: 12日
  - 4年: 13日
  - 5年: 14日
  - 6年以上: 15日

#### 自動計算
- ユーザーの入職日から経験年数を算出
- マスタに基づいて初期有給日数を付与
- 有給申請するたびに、**病院ごとのデータに対して** 残日数がリアルタイムで減少

### 3. カレンダー機能（最重要）

#### スタッフ画面での表示
- **自分の申請**: 編集・削除可能、カレンダーに「有」「希」で表示
- **他人の申請**: 同じ病院内の他のスタッフのみ「〇名希望あり」と数字のみ表示
- **3人以上の申請**: 数字を**赤色**で強調表示
- **他の病院のデータ**: 完全に見えない

#### 管理者画面での表示
- **マトリックス表示**: 同じ病院コード内のスタッフ名と各日の申請が一目でわかる
- **全スタッフの情報**: 同じ病院のスタッフのみ表示
- **他の病院のデータ**: 完全に見えない

#### 申請種別
- **希望休**: 通常の希望休
- **有給**: 有給休暇（残日数の範囲内で申請可能）

### 4. データ永続化と分離

- すべてのデータはブラウザの LocalStorage に保存
- **💡 病院コードごとにフィルタリング**: 各病院はアクセス権限のあるデータのみ表示
- ページリロード後もデータが保持される
- 複数デバイスからアクセスする場合は、各デバイスで独立したデータが保存される

## 🔒 セキュリティに関する注意

このMVPは、**複数の病院が同一システムを利用する場合のセキュリティを考慮した設計**になっています。以下の点に注意してください：

### 実装済みのセキュリティ対策
1. **病院コードベースのデータ分離**: 
   - すべてのユーザーを病院コードで分離
   - すべての休暇申請を病院コードで分離
   - 各病院の有給マスタを独立して管理

2. **ユーザー認証の強化**:
   - 病院コード + ID + PIN（4桁）の3要素認証
   - 病院コード + ID で一意のユーザー識別

### 本番環境で推奨される追加対策
1. **認証**: より強力な認証(OAuth、パスワード、多要素認証など)の導入
2. **データ保護**: バックエンドサーバーおよび暗号化の実装
3. **通信**: HTTPSの必須化
4. **病院コード**: 事前に発行された病院コードと紐付けたユーザー承認フロー（メール認証、管理者承認など）
5. **複数病院の隔離**: 病院ごとの独立したデータベースの構築
6. **ログ・監査**: 全操作のログ記録とアクセス監査
